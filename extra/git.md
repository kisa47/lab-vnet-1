# Система контроля версий Git

Git — это распределённая система контроля версий, которая позволяет отслеживать изменения в коде, работать в команде и управлять проектами. Вот основные команды, разделённые по категориям.

## Инициализация и клонирование репозиториев 

| **Команда** | **Описание** |
|---|---|
| `git init` | Создать новый репозиторий в текущем каталоге |
| `git clone [url]` | Клонировать удалённый репозиторий на локальную машину |

## Работа с изменениями 

| **Команда** | **Описание** |
|---|---|
| `git status` | Показать состояние рабочего каталога и индекса |
| `git add [файл]` | Добавить изменения в индекс (подготовить к коммиту) |
| `git add .` | Добавить все изменения в индекс |
| `git commit -m "Сообщение"` | Зафиксировать изменения с комментарием |
| `git commit --amend` | Изменить последний коммит (например, исправить сообщение) |

## Просмотр истории 

| **Команда** | **Описание** |
|---|---|
| `git log` | Показать историю коммитов |
| `git log --oneline` | Краткая история (по одному коммиту на строку) |
| `git diff` | Показать изменения, которые ещё не добавлены в индекс |
| `git diff --staged` | Показать изменения, добавленные в индекс |
| `git show [hash]` | Показать детали конкретного коммита |

## Работа с удалёнными репозиториями 

| **Команда** | **Описание** |
|---|---|
| `git remote -v` | Показать список удалённых репозиториев |
| `git remote add origin [url]` | Добавить удалённый репозиторий с именем origin |
| `git push origin [ветка]` | Отправить изменения в удалённый репозиторий |
| `git pull origin [ветка]` | Получить и объединить изменения из удалённого репозитория |
| `git fetch origin` | Получить изменения, но не объединять их |

## Работа с ветками 

| **Команда** | **Описание** |
|---|---|
| `git branch` | Показать список веток |
| `git branch [имя]` | Создать новую ветку |
| `git checkout [ветка]` | Переключиться на ветку |
| `git checkout -b [имя]` | Создать и переключиться на новую ветку |
| `git merge [ветка]` | Объединить указанную ветку с текущей |
| `git branch -d [ветка]` | Удалить ветку (если она уже слита) |

## Отмена изменений 

| **Команда** | **Описание** |
|---|---|
| `git reset --hard` | Отменить все изменения и вернуть к последнему коммиту (осторожно!) |
| `git reset --soft HEAD~1` | Отменить последний коммит, но оставить изменения в индексе |
| `git revert [hash]` | Создать новый коммит, отменяющий указанный коммит (безопасно) |

## Дополнительные команды 

- `git stash` — временно сохранить изменения, чтобы переключиться на другую задачу.
- `git stash pop` — вернуть сохранённые изменения.
- `git blame [файл]` — показать, кто и когда изменял каждую строку файла.
- `git cherry-pick [hash]` — применить отдельный коммит из другой ветки.

## Пример типичного рабочего процесса 

```bash
# 1. Клонируем репозиторий
git clone https://github.com/user/project.git

# 2. Переходим в папку проекта
cd project

# 3. Создаём новую ветку для задачи
git checkout -b feature/login

# 4. Вносим изменения
# ...

# 5. Добавляем изменения в индекс
git add .

# 6. Делаем коммит
git commit -m "Добавил форму входа"

# 7. Отправляем изменения в удалённый репозиторий
git push origin feature/login
```
