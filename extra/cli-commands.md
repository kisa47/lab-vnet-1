## Пути в файловой системе линукс 

### Основные пути в файловой системе Linux 

| **Путь** | **Описание** |
|---|---|
| `/` | Корень файловой системы. Все остальные каталоги находятся внутри него. |
| `/bin` | Основные системные команды **не требующие** для выполнения привелегии суперпользователя (например, `ls`, `cp`, `mv`). |
|`/sbin` |Основные системные команды **требующие** для выполнения привелегии суперпользователя (например, `mount`, `lsmod`, `adduser`). |
| `/etc` | Конфигурационные файлы системы и программ. |
| `/home` | Домашние каталоги пользователей (например, `/home/user`). |
| `/root` | Домашний каталог суперпользователя (root). |
| `/usr` | Пользовательские программы и данные. |
| `/usr/bin` | Дополнительные пользовательские команды и программы **не требующие** для выполнения привелегии суперпользователя. |
| `/usr/sbin` | Дополнительные пользовательские команды и программы **требующие** для выполнения привелегии суперпользователя. |
| `/usr/local` | Программы, установленные локально администратором. |
| `/var` | Переменные данные (логи, очереди печати, базы данных и т.д.). |
| `/var/log` | Файлы системных и программных журналов (логов). |
| `/dev` | Файлы устройств (например, диски, терминалы). |
| `/tmp` | Временные файлы, очищаются при перезагрузке. |
| `/proc` | Виртуальная файловая система, содержащая информацию о процессах и ядре. |
| `/sys` | Виртуальная файловая система для взаимодействия с ядром и устройствами. |
| `/media` | Точки монтирования для съёмных носителей (флешки, диски). |
| `/mnt` | Временные точки монтирования файловых систем. |

### Примеры путей 

- **Абсолютный путь:**  
  `/home/user/Документы/отчёт.txt`
- **Относительный путь:**  
  `Документы/отчёт.txt` (если вы находитесь в `/home/user`)
- **Путь относительно домашней папки:**  
  `~/Документы/отчёт.txt`

### Специальные обозначения 

- `.` — текущий каталог
- `..` — родительский каталог
- `~` — домашняя директория пользователя

### Основные команды для работы с путями 

- `pwd` — показать текущий каталог
- `ls [путь]` — показать содержимое каталога
- `cd [путь]` — перейти в указанный каталог

## Команды в Linux 

### Работа с файлами и каталогами 

| **Команда** | **Описание** |
|---|---|
| `ls` | Показать содержимое директории |
| `cd [директория]` | Перейти в указанную директорию |
| `pwd` | Абсолютный путь к текущей директории |
| `mkdir [имя]` | Создать новую директорию |
| `rmdir [имя]` | Удалить пустую директорию |
| `rm [файл]` | Удалить файл |
| `rm -r [директория]` | Удалить директорию с содержимым |
| `cp [источник] [назначение]` | Копировать файлы или директории |
| `mv [источник] [назначение]` | Переместить или переименовать файл/директорию |
| `touch [файл]` | Создать пустой файл или обновить время изменения |

### Чтение и редактирование файлов 

| **Команда** | **Описание** |
|---|---|
| `cat [файл]` | Вывести содержимое файла |
| `less [файл]` | Постраничный просмотр файла |
| `head [файл]` | Показать начало файла (по умолчанию 10 строк) |
| `tail [файл]` | Показать конец файла (по умолчанию 10 строк) |
| `nano [файл]` | Редактировать файл в простом редакторе |
| `vim [файл]` | Редактировать файл в продвинутом редакторе |

### Поиск файлов и текста 

| **Команда** | **Описание** |
|---|---|
| `find [путь] -name [шаблон]` | Найти файлы по имени или другим критериям |
| `grep [шаблон] [файл]` | Искать текст по шаблону в файлах |

### Права доступа и владельцы 

| **Команда** | **Описание** |
|---|---|
| `chmod [права] [файл]` | Изменить права доступа к файлу/каталогу |
| `chown [пользователь]:[группа] [файл]` | Изменить владельца файла/каталога |

В Linux права на файлы и каталоги задаются с помощью команды `chmod`. Она позволяет изменять права доступа для владельца, группы и остальных пользователей.

- **Владелец (user)** — пользователь, которому принадлежит файл.
- **Группа (group)** — группа пользователей, которым назначены права.
- **Остальные (others)** — все остальные пользователи системы.

#### Способы задания прав 

##### 1. Символьный способ (буквенный) #{commands_permissions_litteral}

Используются буквы:
- `u` — владелец (user)
- `g` — группа (group)
- `o` — остальные (others)
- `a` — все (all)

Операции:
- `+` — добавить право
- `-` — убрать право
- `=` — установить ровно эти права

Права:
- `r` — чтение (read)
- `w` — запись (write)
- `x` — выполнение (execute)

Примеры:

| **Команда** | **Описание** |
|---|---|
| `chmod u+x file.sh` | Добавить право на выполнение владельцу |
| `chmod g-w file.txt` | Убрать право на запись у группы |
| `chmod o=r file.txt` | Установить только чтение для остальных |
| `chmod a+rwx file.txt` | Дать всем полные права |
| `chmod u=rwx,g=rx,o=r file.txt` | Задать разные права для всех категорий |

#### 2. Восьмеричный (числовой) способ 

Каждое право имеет числовое значение:
- `r` = 4
- `w` = 2
- `x` = 1

Права задаются трехзначным числом:
- Первая цифра — владелец
- Вторая — группа
- Третья — остальные

Основные комбинации:

| **Права** | **Описание** |
|---|---|
| `755` | Владелец: rwx, Группа: r-x, Остальные: r-x |
| `644` | Владелец: rw-, Группа: r--, Остальные: r-- |
| `700` | Владелец: rwx, Группа: ---, Остальные: --- |
| `777` | Все: rwx (не рекомендуется для безопасности) |

примеры:

| **Команда** | **Описание** |
|---|---|
| `chmod u+x script.sh` | Дать владельцу право на выполнение скрипта |
| `chmod 644 index.html` | Установить права 644 для файла |

#### Просмотр текущих прав 

```bash
ls -l file.txt
```
Пример вывода:
```
-rwxr-xr-- 1 user group 1234 окт 6 12:00 file.txt
```
Расшифровка:
- `rwx` — владелец (чтение, запись, выполнение)
- `r-x` — группа (чтение, выполнение)
- `r--` — остальные (только чтение)

#### Работа с процессами

| **Команда** | **Описание** |
|---|---|
| `sudo [команда]` | Выполнить команду с правами суперпользователя |
| `ps aux` | Показать список процессов |
| `top/htop` | Мониторинг процессов в реальном времени |
| `kill [PID]` | Завершить процесс по его идентификатору |

#### История команд 

- `history` — показать историю команд
- `!номер` — выполнить команду из истории по номеру

## Работа с переменными в Linux (Bash) 

В командной оболочке Linux (чаще всего используется Bash) переменные играют важную роль: они позволяют хранить и использовать данные, передавать параметры между командами и скриптами.

### Создание и присваивание значения 

Для создания переменной просто напишите её имя, поставьте знак `=` и сразу за ним — значение (без пробелов!).

```bash
name="Ivan"
count=10
path=/home/user/docs
```

**Важно:** Между именем переменной, знаком `=` и значением не должно быть пробелов.

### Использование переменных 

Чтобы получить значение переменной, перед её именем поставьте знак `$`.

```bash
echo $name      # Выведет: Ivan
echo "Количество: $count"  # Выведет: Количество: 10
cp file.txt $path/          # Скопирует файл в /home/user/docs/
```

### Встроенные переменные оболочки 

В Bash есть специальные переменные, которые доступны по умолчанию:

| **Переменная** | **Описание** |
|---|---|
| `$0` | Имя текущего скрипта |
| `$1, $2, ...` | Позиционные параметры (аргументы скрипта) |
| `$#` | Количество переданных аргументов |
| `$@` | Все аргументы как отдельные слова |
| `$*` | Все аргументы как одна строка |
| `$$` | PID (идентификатор) текущего процесса |
| `$?` | Код возврата последней команды (0 — успех, не 0 — ошибка) |
| `$USER` | Имя текущего пользователя |
| `$HOME` | Домашняя директория пользователя (как `~`) |
| `$PWD` | Текущая директория (как команда `pwd`) |
| `$PATH` | Список директорий, которые используются для поиска исполняемых файлов |

### Экспорт переменных 

Если переменную нужно передать в дочерние процессы (например, другим скриптам или программам), используйте команду `export`.

```bash
export API_KEY="secret123"
# Теперь этот ключ доступен во всех командах, запущенных из этого терминала
```

Или можно объединить объявление и экспорт:
```bash
export PATH=$PATH:/usr/local/bin
```

### Удаление переменных 

Чтобы удалить переменную, используйте команду `unset`.

```bash
unset name
echo $name      # Выведет пустую строку
```
